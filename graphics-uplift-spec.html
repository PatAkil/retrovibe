What the review changed

Two adversarial experts (game-feel/player-experience and retro-rendering/performance) critiqued the plan each round; an arbiter accepted only real, non-duplicate improvements, folded in by a reviser. 69 improvements over 6 rounds. Headline resolutions: aberration is now strictly opt-in/off (was "on-but-subtle"); a byte-identical golden-frame gate proves the default path is unchanged; contrast is re-measured on the final post-CRT frame; a reduced-motion + photosensitivity safety system spans all new effects; glow drops ctx.filter for a cheap radial sprite + bilinear-halving blur; aberration is a capped uniform split delivered via new crt.pulse / setFrozen / drawOverlay APIs.

The architectural fact that shapes everything

Every game carries its own copy of engine/ — the modules are cloned with the template, each game standalone. So "extend the shared engine" means editing the template's engine/.

That change reaches newly created games only. The 8 games already in workspace/ keep their frozen engine copies and are untouched — zero regression surface, and validation narrows to a single throwaway clone.

01

Guardrails

Backward-compatibility & the golden gate

  • Additive only, every new effect opt-in / default-off. Existing exports keep their exact signatures; new behavior arrives via new optional params + two new modules. An unset option — including aberration — produces a byte-identical frame to today.
  • New gateByte-identical golden regression (one-time, gates the whole change). check/build/smoke/eyeball only prove it compiles, boots, and looks fine — not that the default path is unchanged. Render N deterministic frames from the current template, hash each backing store; after the edits, render the same frames with all new options unset and assert the hashes match. Determinism requires three harness overrides, because juice.preRender/particles call Math.random() directly: (a) seed Math.random, (b) pin matchMedia('(prefers-reduced-motion)'), (c) drive a scripted dt so flicker's clock is deterministic. Flicker stays on clock += dt and crt.pulse (the only new performance.now() consumer) is unset on the default path, so no performance.now mock is needed. Assert under both matchMedia states.

Validation — the eyeball step must be moving

  • Throwaway clone per workstream. Template never run in place: cp -r game-template → scratch game → check + build + smoke green, then the per-workstream checks below. Discard.
  • In-motion readability. Drive the tracked actor across each new surface (shaded sprite over background, star field, grid, glow region) at gameplay speed and confirm the crisp read holds — never a frozen frame.
  • Composited-buffer contrast, on the final post-CRT frame. Glow/shading composite before crt.render, which darkens alternate rows and vignettes the edges — a sprite can pass pre-CRT yet fail on-screen at the periphery. Sample the actual composited-and-CRT'd buffer over the peak-glow / grid-crossing / densest-star region on the worst-case bright frame, including a dark-scanline row at a vignette edge, and require actor contrast ≥ 3.0. For glow this is an offline two-sample ratio, not one luminance.
  • Colorblind + HUD legibility. Red-green check on colored stars and any gameplay-meaning glow color. Confirm the 3×5 HUD, title text, and one in-world popup (GET READY / PAUSED / score-combo) stay legible at the shipped aberration cap and over the brightest surface they can overlap, including the stacked multi-line HUD clear of SAFE_MARGIN.

Contrast, HUD-over-bright-layers

  • The contrast gate still rules — measured on the composited post-CRT buffer. Colored stars stay in the 1.8–2.5:1 ambient band (luminance only; chroma/hue constrained in WS2); grid lines and glow may not drop any actor below 3.0:1, measured on the real frame, not the flat palette. ensuring-arcade-visuals §1b amended, not bypassed.
  • RequiredHUD text vs the new bright layers. The three new bright layers share the HUD corners, and thin 1px text is never sampled by the actor-pixel checks. Either (a) measure drawScore/drawLives/hudText contrast on the composited post-CRT frame against the brightest corner surface (glow bleed / near-grid rows / dense star clusters) and require the read holds, or (b) reserve a HUD safe-zone backgrounds/glow may not brighten (glow masked to zero within SAFE_MARGIN bands, grid horizon below the HUD rows). State the chosen resolution beside §1b.

Reduced-motion & photosensitivity — one damper across all new effects

  • New systemA single intensity lever, defaulting to prefers-reduced-motion at startup. Dampens the stacked shimmer/motion load (grid-scroll + parallax + flicker + bloom + aberration) together, across three categories:
    • Ambient / decorative (grid-scroll, parallax, flicker, steady aberration, decorative bloom) — dampened.
    • Gameplay telegraphs — exempt: the damper may not zero a telegraph's urgency channel (pulse/blink), since a reduced-motion + colorblind player would lose both differentiators. Exempt them, or require a static reduced-motion-safe alternate cue.
    • Impact-feedback transients (crt.pulse, co-firing impact bloom) — their own category, dampened (the crisp flash/shake still convey the hit).
  • PAUSED-state ambient freeze. On pause the world halts but crt.render and the new ambient layers keep running — freeze or visibly slow grid-scroll, parallax drift, and decorative bloom so "paused" reads as paused and eye-strain stops accumulating.
  • WCAGCombined-transient photosensitivity ceiling. The plan stacks three full-screen brightness spikes on major events (existing death-flash + aberration pulse + impact bloom). When they co-fire, cap the aggregate luminance step and enforce a minimum interval, so rapid major events can't produce a flash train.
  • The damper never touches the pre-existing flicker baseline. Default-on applies to new opt-in effects only — a reduced-motion machine running a fresh clone with all options unset must render today's flicker, or the byte-identical promise breaks.
  • No commits during the loop. Engine edits stay uncommitted until you explicitly ask to save.

Workstreams — low-risk → high-risk

1

Gradient & shaded sprites

engine/draw.ts · palette.tsRisk · Low

Shade at makeSprite time, not per-frame — precompute a per-row ramp so drawSprite's hot loop is untouched and perf is neutral. New optional arg { ramp?: number[], band?: number }. Solid bands only in v1 — dithering-between-steps deferred to a per-sprite opt-in (it fights scanlines/motion/aberration on small sprites). Unset = byte-identical.

  • ResolvedRamp semantics — per-palette shade ladders, never an RGB lerp. Absolute palette-index steps and "tint each map color per row" are incompatible for a multi-color row, and makeSprite is genuinely multi-color. Locked: each palette color declares its own darker-by-N chain (a shade ladder) in palette.ts; the ramp means "N steps down each color's own ladder," which preserves multi-color rows and stays on-palette. A free {top,bottom} lerp (off-palette) is forbidden. Fallback if a color has no ladder: that color's cells render flat.
  • Resolved · Option APer-palette ladder feasibility — audit, don't assume. Grounded in the actual palettes: ramp palettes (DUSK/SUNSET/OCEAN/GAMEBOY) are ordered dark→light, so ladders come nearly for free — but depth is capped by the darkest-step 3.0 floor (a mid-ramp actor is ~1–2 bands before it reaches background luminance) and the hue-drift check. PICO8 is 16 distinct hues, not a ramp: white/green/blue/orange have usable darker siblings, but red/pink/yellow are marginal (drift to purple/orange). Locked (A): author ladder metadata per palette where it clears both floors, degrade-to-flat elsewhere (never silently), and demonstrate the reference shading on a ramp palette (SUNSET/OCEAN) — no palette constants change. Appending dark siblings to PICO8 is a possible fast-follow, not v1.
  • Fixed-orientation only. A baked top-lit fade is wrong-lit on a rotated/mirrored/multi-facing actor. Expose a ramp axis alignable to facing, or keep rotating actors flat / re-bake per orientation. Caveat goes in the skill.
  • Darkest-step + actor-vs-actor floors. The darkest ramp/band step clears 3.0:1 vs every overlapped surface, and gradient depth is bounded so lower rows don't dissolve. Also: two actors distinct as flat colors can converge to near-identical dark bottoms — validate lightness/hue separation at every band, not just against the background.
  • Moire in one coordinate space. The CRT scanline period is 2 logical px, so band steps must be a multiple of 2 logical px. At PX≥2 a per-row band already aligns; the risk is concentrated at PX=1, where the banded path is mutually exclusive with strong scanlineAlpha.
2

Background layers

engine/background.ts · particles.tsRisk · Low

createGrid({ color, horizon, spacing, scroll }) renders a Tron-style receding floor (palette-indexed geometry, behind the world); createParticles gains ambientColors?: string[] for a per-particle hue subset. Single-color path unchanged.

  • Genre-gated, single depth metaphor. A scrolling grid/starfield belongs to scrolling/endless genres; a fixed-arena game gets static or drift-only (a scrolling background asserts false vection). Never combine a receding ground-grid with a deep-space starfield — contradictory depth cues; pick one.
  • Grid in the contrast gate + no false targets. Grid line color runs through contrast() vs every actor (≥3.0 where they overlap); bound scroll speed and line brightness. Cap intersection-node brightness ≤ line brightness (additive crossings become point-like false targets like near-layer stars), or draw flat lines with no emphasized nodes.
  • Integer-logical-step scroll is the default. Draw 1-logical-px lines quantized to integer logical steps — pixel-pure, frame-stable, matching the engine's pixel identity. A device-space pass (for dense receding lines) is an explicit opt-in only, and must carry the relative-scale shake mechanism.
  • Device-space must preserve shake. Use a relative ctx.scale(1/scale,1/scale) inside the shake window (composes with the shake translate) — never setTransform, which discards shake and leaves a rock-still grid. Decide and document whether the background shakes fully, parallax-damped, or not at all.
  • Colored stars = a subtle hue wash. ambientColors are low-saturation near-grey tints with small hue delta (the band governs luminance only); cap near-layer size/speed/brightness below actor salience; red-green-safe subset. Each particle's hue is fixed once at spawn and stored — never re-picked per frame (that flickers). Parallax drift freezes on PAUSED.

Glow / bloom pass

engine/glow.tsRisk · Med

Games draw bright things into an offscreen (glow.ctx); glow.composite(pc.ctx) blurs and composites additively, in world space inside the shake window, before juice.postRender. Opt-in.

  • PerfTwo tiers, no ctx.filter anywhere. Default (a) cheap radial sprite: precompute one soft radial-gradient alpha sprite, blit additively — zero per-frame filter cost, covers the img2 range circle. (b) Resample blur for arbitrary edges: downscale the logical buffer in successive halvings (240→120→60, imageSmoothing on) then one upscale — the resample passes are the blur. The 2×-per-step cap is required: a single 4× jump point-samples and a moving source shimmers.
  • Never blooms a tracked actor. Glow renders behind the crisp sprite (drawn sharp on top) and is scoped to telegraphs, decorative accents, projectiles, or self-glow — never under a different actor the player must track. Additive blur on tracked edges dissolves the crisp read exactly when positioning matters.
  • Gameplay boundaries need a crisp ring. Any glow that communicates a boundary (range circle, blast radius, kill zone) pairs the soft halo with a crisp 1px palette-indexed ring at the true edge — the bloom is decoration, the hard line is the contract.
  • Impact bloom is a transient, not wallpaper. Snappy attack + short punchy decay (~0.1–0.2s, matching juice.flash/crt.pulse), decoupled from ambient glow. Frozen-hold: while juice.frozen, hold source/intensity and begin decay only after release — particles.update runs before the frozen gate, so a naive bloom drifts through the emphasized freeze.
  • Contrast by design, not readback. Budget per-source alpha/brightness, cap the composite globalAlpha, design-limit worst-case overlap count. A per-frame getImageData clamp is a multi-ms stall and disallowed — and 'lighter' clamps at 255 anyway, so only design-time budgeting recovers contrast.
  • Authoring CTM contract. Author glow with identical drawSprite calls (same px). The buffer's space is logical 240×160; state canvas.width/height per option (full-res 720×480 with baked scale-3, or half-res 120×80 with baked scale/6 + smoothing). Composite under the ambient CTM — never reset to identity. Assert px=3 lands pixel-identical and the halo registers with an active shake.
  • State & transform correctness. Wrap the whole composite (and WS4's pass) in one save()/restore() — covers imageSmoothingEnabled, which the resample flips on. Clear the buffer every frame. Exit invariant on restore: filter='none', op='source-over', alpha=1, imageSmoothingEnabled=false — a leaked 'lighter' makes the death flash additive and neutralizes CRT's black scanlines (black+lighter = no-op), silently disabling the filter. Colorblind telegraphs also encode urgency on a non-hue channel (pulse/size/blink), exempt from the reduced-motion damper. Off-palette exemption for glow (like vignette/flash/CRT) added to the skill.
4

CRT chromatic aberration

engine/crt.tsRisk · High

CrtOptions.aberration, strictly opt-in / default-off (supersedes the old on-but-subtle recommendation). A capped uniform device-px channel split — the fiddliest module, hence last.

  • Uniform, not radial; capped for fairness. A drawImage translate is spatially uniform; a true radial split needs fractional sampling + smoothing (contradicts the whole-device-px, smoothing-off basis). Hard-cap peak split < ~1 effective device px so a 1px projectile's ghosts still overlap its collision cell. Don't raise to 2px on "gameplay layers" — the split can't segregate layers. Validate actor and projectile across a high-split zone: apparent position == hitbox.
  • Steady and pulse are mutually exclusive per game. A whole-px offset under a <1px cap admits only 0 or 1, so a nonzero steady split consumes the whole budget and leaves crt.pulse adding 0 visible px. Model: steady default = 0; the pulse owns the single 0→1px transient step. A game picks one.
  • APIUn-split HUD + all bitmap text. Add an optional drawOverlay callback to crt.render, invoked after the aberration pass but before scanlines, with the baked logical transform re-established — the HUD paints clean. Games must move HUD draws into it. Title text and in-world popups are aberrated too (3×5 strokes are 1 logical / 3 device px → ~33% fringing), so route all bitmap text through the overlay, or forbid aberration on text-primary scenes.
  • True channel split onto black. Snapshot the finished frame to a scratch offscreen, clear main to black, then per channel: source-over copy snapshot into a reusable chanBuf, multiply the R/G/B mask, lighter-composite at the capped offset. Reset to source-over at the top of channels 2 & 3 or the opaque snapshot redraws multiplied and corrupts them. Additive ghosting of the full-RGB frame merely doubles brightness — not a split.
  • Lazy one-time allocation; honest op budget. createCrt has no canvas ref and render gets only logical w/h, so allocate lazily on first render keyed on device dims — snapshot scratch, single chanBuf, radial halo sprite, baked vignette gradient. Per frame ≈ snapshot + clear + ~3 channels (copy + mask + offset-composite each) ≈ 8–11 device-res ops.
  • Order + clocks. Aberration runs on the world/flash frame first, then drawOverlay, then scanlines/vignette/flicker un-aberrated (the "glass," not the signal) to avoid colored moire. Leave flicker on clock += dt; route only crt.pulse through performance.now() — switching flicker off accumulated-dt is a default-path behavior change that breaks the golden hash.
  • APIJuice tie-in — crt.pulse + crt.setFrozen. Add crt.pulse(mag, durationSeconds) (decaying transient added to the steady split, same cap), called from the same branch as juice.shake/flash, with real wall-clock decay. Add crt.setFrozen(bool) (crt has no juice ref) to pause the pulse's decay during hit-stop so the most-emphasized frozen instant doesn't de-aberrate. Reserve pulse for major events, mirroring juice's shake≥4–6px / flash≥0.3s floor.
03b

Perf gate (Workstreams 3 & 4)

  • ExplicitMeasure margin, not just the cliff. Primary metric: sustained requestAnimationFrame inter-frame delta over a few seconds with effects animating, counting dropped frames (>~20ms), in headed Chromium with GPU raster, running the real pipelines (actual glow composite + actual channel-split — not a stubbed offset). Synchronous performance.now() around render() is a secondary JS-cost signal only — it misses deferred GPU raster cost.
  • Stress ramp for headroom. rAF delta clamps at vsync, so dropped-frame counting only catches regressions already over budget. Scale glow-source / particle / grid density upward until the first sustained dropped frame; report that breaking density as the margin. Feasibility: confirm the gate can run headed-with-GPU here (PLAYWRIGHT_BROWSERS_PATH is set — likely headless/SwiftShader), or name the concrete machine it runs on.
  • Dedicated stress fixture, not the sparse reference game. Max-density glow at the worst-case overlap region + a full grid + both parallax layers at max count + aberration on. Report sustained rAF delta, dropped-frame count, and ramp-to-failure density. State the current template baseline delta so the budget is a measured delta from it, and re-confirm against WS4's ~8–11 op count.
